## 💎 デザインシステムの定義

### 1. カラーパレット & スタイル
* **背景 (Background)**: ディープネイビーとダークインディゴのグラデーション (`#0B0F19` から `#162032`)。
* **カード・コンポーネント (Surfaces)**: 背景を半透明に透過させたグラスモフィズム（アクリル調）質感 (`rgba(22, 32, 50, 0.45)`、`backdrop-filter: blur(12px)`)。淡い枠線 (`rgba(255, 255, 255, 0.08)`) を使用。
* **プライマリカラー**: エレクトリック・インディゴ (`#5E5CE6`)。ボタン、アクティブタブ、主要なフローアークに使用。
* **アクセントカラー**: ネオン・ミント (`#30D158`)。適合度（マッチスコア）やアクティブステータス、採択組織ノードに使用。
* **タイポグラフィ**: ヘッダーは `Outfit`、本文は `Inter` を適用し、情報の可読性と洗練さを両立。

## 🧭 メインナビゲーション & ビュー切り替え

画面上部に固定されたグローバルナビゲーション、または左側サイドバーにより、以下の4つのビューをシームレスに切り替えます。

1. **Dashboard (Overview)**: 全体サマリーとスケジュール管理。
2. **Money Flow (Visualizer)**: 予算から助成金へのマクロ・ミクロな資金移動の可視化。
3. **Applications (List & Details)**: 助成金一覧とAI適合判定、申請書自動生成。
4. **Settings**: 団体プロファイルやAPIキーの設定。

## 🖥️ 各ビューのUI/UX詳細

### 1. Dashboard (Overview) ビュー
* **📅 予算カレンダータイムライン (Budget Calendar Timeline)**
  * **概要**: 国の予算成立・概算要求マイルストーンや、個別助成金の申請締切をガントチャート形式で直感的に把握できるエリア。
  * **インタラクション**: マイルストーンや申請予定ホバー時にポップオーバーで詳細概要、関連省庁、申請金額を表示。
* **🎯 新着・推薦助成金 (Recent Subsidy Opportunities)**
  * **概要**: クローラーが取得した最新データに、AIが算出した適合度マッチスコア（例: `95%`）をネオン・ミントのプログレスリングとしてカード上に表示。
* **🧠 AIエージェントステータス (Modal AI Agent Status)**
  * **概要**: Modal上のQwen3/BgeRerankerの稼働状況（ウォームアップ中、スキャン完了など）をリアルタイムに進捗バーとログで示すウィジェット。

### 2. Money Flow (Visualizer) ビュー
* **📊 サンキーダイアグラム (Sankey Flow)**
  * **可視化チェーン**: `国庫 (一般会計等) ──> 省庁 ──> 予算事業 ──> 【個別助成金公募】 ──> 【最終受給者 (企業/NPO)】`
  * **UX**: ノードをホバーするとそのノードに接続するエッジ（資金フロー）がハイライトされ、流れる金額（JPY）と構成比が表示される。
  * **ドキュメント参照**: 助成金ノードをクリックすると、右側サイドパネルにその助成金の詳細情報と申請画面への導線が出現する（Flow-to-document back references）。
* **🌐 3D地球儀 (3D Globe)**
  * **概要**: `react-globe.gl` を使い、国同士の貿易や国際的・広域自治体間の資金移動を地球儀上に光る3Dアーク（放物線）として描画。ドラッグによる回転、ホイールによるズームが可能。

### 3. Applications (一覧 & 詳細) ビュー
* **📋 助成金一覧テーブル**
  * マッチスコア順、締切順、金額順でソート可能なデータグリッド。
  * 自治体（富山市・富山県）や省庁（経済産業省・こども家庭庁など）による即時フィルタリング。
* **🔍 AI適合判定・エビデンス詳細（モーダル）**
  * **要件適合判定テーブル**: 助成金募集要項から抽出した要件（売上制限、従業員数、事業目的など）と、団体プロファイルをAIが突き合わせ、「適合」「要検証」「不適合」をシグナル表示。
  * **ハルシネーション対策（直接引用と出典表示）**: AIの判定根拠セル内に、**「募集要項の直接引用テキスト (Quote)」と「出典 (URL、PDFのページ番号)」を紐付けて表示**し、ユーザーがワンクリックで一次ソースを確認可能にする。
  * **期待値評価レーダーチャート**: 「金額効率」「採択見込み」「書類負担」「戦略整合性」の4軸でAIがスコアリングした結果を可視化。
  * **申請書プレビュー・ダウンロード**: 自動記入された `*_filled.xlsx` のプレビューと、原本テンプレートを並べてダウンロード可能。

## 🔗 現状の仕様・ワークフローとの統合設計

本 UI/UX デザインは、標準スキル `grant_application_workflow` で定義されている Phase 0 〜 Phase 7 の各手順およびシステムフローを可視化・操作するために設計されています。

| ワークフローフェーズ | UI/UX での具体的な統合・表現方法 |
|---|---|
| **Phase 0-1 (登録と収集)** | ・新着収集された補助金が `Recent Subsidy Opportunities` カードに自動追加。<br/>・クローラーのバッチ実行や進捗が `Modal AI Agent Status` に連携表示。 |
| **Phase 2 (RAG適合判定とエビデンス)** | ・補助金カードの「詳細表示 (View Details)」から、ハルシネーション防止ルールに基づく**「募集要項の直接引用 (Quote)」と「出典」を紐付けた適合性判定テーブル**をモーダル表示。 |
| **Phase 3 (期待値評価)** | ・`MATCH SCORE`（ミント色のプログレスリング）の裏付けとして、「金額効率」「採択見込み」「書類負担」「戦略整合性」の4軸レーダーチャートを表示。 |
| **Phase 4-5 (計画と下書き作成)** | ・`Applications` ビューから、保護された「原本テンプレート」と、AIが自動記入した「`*_filled.xlsx` 複製ファイル」をワンクリックでダウンロード・プレビュー可能。 |
| **Phase 6 (進捗連携)** | ・`Budget Calendar Timeline` において、国の予算マイルストーンと、`ai-note-meet` から登録された具体的な申請タスク・カレンダー予定をガントチャート上にマージして表示。 |
| **Phase 7 (提出前検証・プリフライト)** | ・申請書のエクスポート画面で、`grant_submission_preflight_run` の実行プログレスと、エラーや入力漏れ（カバレッジ不足）がある場合の警告シグナルを分かりやすく提示。 |
